The problems that responsive web design solves

Responsive web design (RWD) solves the following problems:

  • Problem: Only a subset of people (devices) can access the site. With RWD, the greatest number of people can access the site.
  • Problem: There are too many different devices to design a site for. RWD supports many types of devices with a single, flexible design using breakpoints and flexible grids. There are too many devices to target specifically, especially mobile touchscreen devices, which have a different user experience than laptop or desktop computers with keyboard and mouse. Game consoles with a game controller provide yet another user experience. Back when there were laptop screens and small smartphone displays, it seemed to make sense to target 2 or 3 specific devices. But there are too many devices now, some with multiple display orientations, so it makes the most sense to sense the current viewport width and then place that within a given range, and define layouts for a given page dynamically, based on the current viewport width.
  • Problem: Non-mobile-friendly sites rank lower in search results. For SEO, mobile-friendly sites rank higher in search results. RWD makes it easy to provide mobile-friendly sites.
  • Mobile-first website design means using a desktop environment to code a website for use on a mobile touchscreen. RWD bridges this gap, making it easy to use a laptop to code a site mainly for mobile use. Mobile-first helps prioritize the content: when coding a website on a laptop, make the browser window 1/4 of the screen width, and test the result on a mobile device.
  • It is difficult (costly, slow, and complex) to maintain a separate website for each viewport width, as is done with the m-dot approach. In the m-dot approach, a separate version of the web site is created and maintained, for narrow, mobile displays. For example, http://m.amazon.com. This approach is difficult to maintain, and typically resulted in limited functionality, a subset of information, and outdated information.
  • It is cumbersome to serve out different versions of a web page from the server for different viewport widths, as is done in the adaptive-design approach. In the adaptive design approach, the server takes in the width of the display, and then serves out a version of the page that targets that width.

Responsive web design dynamically lays out a webpage to fit within displays of various widths on various devices. A flexible grid (fluid grid) is typically used, providing proportional, relative sizing of columns and rows.

Media queries in CSS define breakpoints, to change the layout when the width of the display (viewport) is within a specified range.
