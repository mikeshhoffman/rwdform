The problems that responsive web design solves

Responsive web design (RWD) solves the following problems:

  • Only a subset of people (devices) can access a non-RWD site. With RWD, the greatest number of people can access the site. Mobile-first website design means using a desktop environment to code a website first for adequate functionality on a mobile touchscreen, and then enhancing the functionality on a laptop.
  • There are too many different devices to design a site for, including mobile touchscreen devices, game consoles with game controllers, and desktop computers with keyboard and mouse. RWD supports many types of devices with a single, flexible design using breakpoints in media queries together with flexible grids, providing proportional, relative sizing of columns and rows.
  • With the m-dot approach, it is costly, slow, and complex to maintain a separate website for a wide and a narrow viewport width. The m-dot approach typically resulted in limited functionality, a subset of information, and outdated information.
  • In the adaptive-design approach, it is cumbersome to serve out different versions of a web page from the server for different viewport widths.
  • Non-mobile-friendly sites rank lower in search results. For SEO, mobile-friendly sites rank higher in search results. RWD makes it easy to provide mobile-friendly sites.
