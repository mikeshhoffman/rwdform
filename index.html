The problems that responsive web design solves

Responsive web design solves the following problems:

  • There are too many devices to target specifically. Back when there were laptop screens and small smartphone displays, it seemed to make sense to target 2 or 3 specific devices. But there are too many devices now, some with multiple display orientations, so it makes the most sense to sense the current viewport width and then place that within a given range, and define layouts for a given page dynamically, based on the current viewport width.
  • It is difficult to maintain a separate website for each viewport width, as is done with the m-dot approach. In the m-dot approach, a separate version of the web site is created and maintained, for narrow, mobile displays. For example, http://m.amazon.com. This approach is difficult to maintain, and typically resulted in limited functionality, a subset of information, and outdated information.
  • It is cumbersome to serve out different versions of a web page from the server for different viewport widths, as is done in the adaptive-design approach. In the adaptive design approach, the server takes in the width of the display, and then serves out a version of the page that targets that width.

Responsive web design dynamically lays out a webpage to fit within displays of various widths on various devices. A flexible grid is typically used.

Media queries in CSS define break points, to change the layout when the width of the display (viewport) is within a specified range.
